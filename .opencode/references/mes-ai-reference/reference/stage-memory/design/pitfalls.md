# design 阶段全局坑点库

## 一、使用说明

1. 本文件仅收录具有跨需求复用价值的设计阶段坑点
2. 每条坑点必须可回溯到来源 REQ 与证据文件
3. 详细上下文保留在来源 REQ 的局部阶段记忆中

---

## 二、坑点条目

### MEM-DESIGN-G-001 设计脱离目标仓真实结构

- 阶段：design
- 类型：pitfall
- 状态：active
- 严重级别：high
- 触发条件：
  - 涉及多服务、多模块或存量系统改造
  - 只依据需求文本或理想架构图设计方案
- 问题表现：
  - 设计文档与真实服务边界、模块边界、包结构不一致
  - 下游开发阶段无法直接映射到目标仓真实实现位置
- 根因：
  - 未先核对初始化产物、code-map、detail、file-summaries 等结构化事实
- 规避要求：
  - 必须先确认真实服务/模块/Schema 归属
  - 必须在设计中标注“复用 / 修改 / 新增”
  - 不得用骨架默认示例路径替代真实路径
- 必检项：
  - `backend-overview.md` / `frontend-overview.md`
  - 相关 `detail.md` / `file-summaries.md`
  - 初始化阶段的路径映射与索引产物
- 来源与证据：
  - 来源需求：后续按实际 REQ 补充
  - 来源阶段：design
  - 证据路径：后续按实际 REQ 补充
