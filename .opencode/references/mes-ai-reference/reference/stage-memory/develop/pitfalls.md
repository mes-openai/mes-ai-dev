## 二、坑点条目

### MEM-DEVELOP-G-001 修 Bug 顺手重构

- 阶段：develop
- 类型：pitfall
- 状态：active
- 严重级别：high
- 触发条件：
  - 以缺陷修复或局部改动为目标
  - 在开发中额外扩大重构范围
- 问题表现：
  - 原始需求范围被放大
  - 回归风险上升，审查与测试成本失控
- 根因：
  - 未坚持最小改动原则
  - 将“顺手优化”误当成低成本动作
- 规避要求：
  - Bugfix 必须最小化修改
  - 若确需重构，必须显式升级范围并补设计/审查依据
  - 阶段记忆中必须记录范围变化原因
- 必检项：
  - task-plan
  - design handoff
  - stage-memory
- 来源与证据：
  - 来源需求：后续按实际 REQ 补充
  - 来源阶段：develop
  - 证据路径：后续按实际 REQ 补充
