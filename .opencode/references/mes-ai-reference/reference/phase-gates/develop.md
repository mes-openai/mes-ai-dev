# 代码开发阶段门禁

## 一、进入门禁

### must-pass
- 详细设计阶段已退出
- 设计文档包含实现所需的 API / 数据模型信息
- 不存在未处理的并发锁冲突

---

## 二、退出门禁

### must-pass
- `tasks.md` 已生成，或已明确与历史 `task-plan.md` 的兼容映射
- `test-cases.md` 已先于代码生成形成，且已记录用户补充区与用户确认结论
- 代码开发阶段完成产物报告已生成，且列出标准产物、已生成文件、文件作用、未生成文件及原因
- 计划内代码文件已完成
- 后端编译通过（如涉及后端）
- 前端构建通过（如涉及前端）
- 本轮新生成测试用例全部通过
- 本轮生成/修改并纳入验证范围的行覆盖率、分支覆盖率、方法覆盖率均达到 100%
- `self-review-report.md` 已生成且无严重未解决问题
- `tasks.md` 未重定义仓级责任边界与 provider 选择
- 本次实现引用的项目私有契约/范式已列出
- 若偏离既有私有契约/范式，理由与影响已记录
- 在存量项目中，代码生成位置、分层、命名、依赖组织与测试组织已对齐目标仓既有结构
- 若结果将进入测试阶段，验证对象变化清单已形成或已说明为何当前不需要
- 阶段评审结论为 ✅通过 / ⚠️有条件通过
- 本轮改动范围与任务目标一致，未包含无关格式化、无关重命名、无关重构或顺手优化

### should-check
- `db-migration.md`、`impact-ledger.md`、`verification-evidence.md`、`refresh-hints.md` 已生成（按需）
- `verify-object-change-list.md` 已生成（按需）
- `tasks.md` 已覆盖 TDD 交接结论，或已明确兼容映射到历史 `task-plan.md`
- 代码符合 `coding-standards.md`
- 设计偏差已留痕
- 已检查编码前思考结果，包含设计结论、任务边界、现有模式、验证目标和风险点
- 已检查简洁优先，优先复用既有分层、工具类、接口、测试组织与已有实现模式
- 已检查精准修改，未触碰当前任务无关文件、模块或知识产物
- 若使用 GitNexus 类能力，已说明其辅助定位的最小改动文件集合、消费者与回归路径；若超出设计边界，已回流 analyze/design

### advisory
- 变更清单与数据回归材料已同步沉淀

---

本阶段的通用步骤级门禁、评审结构与 mandatory-record 要求，统一以 `common.md` 为准。
