# 配置瘦身审计模板

> 本模板用于系统性审查 system / developer / AGENTS / rules / skills / tools 的上下文占用与规则归位情况。
> 适用于瘦身前盘点、瘦身中迁移、瘦身后复盘三个阶段。

---

## 一、审计目标

本审计用于回答以下问题：

1. 这段内容是否必须常驻？
2. 这段内容是否与其他层重复？
3. 这段内容应放在哪一层最合理？
4. 这段内容应该保留全文、摘要化还是迁出？
5. 迁出后应该由哪个索引或细则文档承接？

---

## 二、审计范围

建议按以下来源逐层审查，不混合处理：

- System prompt
- Developer prompt
- `AGENTS.md`
- Rules / Reference 索引页
- Skill 注册表与 Skill 说明
- Tool schema / 工具说明

---

## 三、标签体系

### 3.1 内容类型标签

- `R`：红线 / 硬约束
- `G`：路由 / 入口
- `P`：流程规则
- `E`：示例
- `B`：背景说明
- `S`：结构 / 目录说明
- `T`：工具说明

### 3.2 处理动作标签

- `KEEP`：保留常驻
- `SUM`：摘要化
- `MOVE`：迁出按需加载
- `DROP`：删除重复或低价值内容
- `MERGE`：与其他段合并重写

---

## 四、单段审计卡

对每一段内容填写以下字段：

```md
## 审计卡

- 编号：
- 来源层：System / Developer / AGENTS / Rules / Skill / Tool
- 文件 / 位置：
- 原文摘要：
- 内容类型：R / G / P / E / B / S / T
- 当前作用：
- 使用频率：高 / 中 / 低
- 风险等级：高 / 中 / 低
- 是否重复：是 / 否
- 若重复，主定义位置：
- 建议动作：KEEP / SUM / MOVE / DROP / MERGE
- 迁移目标位置：
- 精简后建议文案：
- 备注：
```

## 五、判定规则

### 5.1 是否必须常驻

逐段回答以下问题：

1. 这段内容是否被 80% 的请求使用？
2. 一旦忘掉这段内容，是否会造成明显风险？
3. 这段内容是否很短但能显著约束行为？
4. 这段内容是否是其他规则的导航入口？

判定建议：

- 满足 2 条及以上：倾向 `KEEP`
- 只满足 1 条：倾向 `SUM`
- 一条都不满足：倾向 `MOVE` 或 `DROP`

### 5.2 是否重复

检查以下问题：

1. 同一规则是否在多个层级重复出现？
2. 是否只是换了说法，本质仍是同一规则？
3. 是否存在“主定义 + 重复展开”的情况？

判定建议：

- 若重复，必须确定唯一主定义层。
- 其他位置只能摘要引用或删除。

### 5.3 是否只是解释而非规则

以下内容通常不应常驻：

- 为什么这样做
- 背景说明
- 示例 A / B / C
- 经验性扩展说明

判定建议：

- 倾向 `MOVE`
- 若有必要，仅保留一句结论做 `SUM`

## 六、分层归属判断表

| 内容 | 推荐层级 |
|---|---|
| 平台级红线 | System / Developer |
| 行为路由 | Developer |
| 项目级铁律 | AGENTS |
| 项目结构与总入口 | AGENTS |
| 阶段规则 | Phase |
| 特殊场景规则 | Scenario |
| 审查 / 产物 / 留痕 | Governance |
| 业务术语 / API / 错误码 | Reference |
| Skill 使用细则 | Skill / Governance |
| 工具长说明 | Tool schema / 外部工具文档 |

---

## 七、整层审计总表模板

```md
| ID | 来源层 | 内容摘要 | 类型 | 频率 | 风险 | 重复 | 动作 | 新位置 |
|----|--------|----------|------|------|------|------|------|--------|
| 01 | AGENTS | 项目概况 | G | 高 | 中 | 否 | KEEP | AGENTS |
| 02 | AGENTS | 长路径枚举 | G | 中 | 低 | 否 | SUM | rules/*/index.md |
| 03 | Developer | 非实现请求不改代码 | R | 高 | 高 | 是 | MERGE | Developer 主定义 |
```

## 八、审计输出建议

审计完成后，至少产出以下结果：

### 8.1 瘦身结果总表

```md
- 保留常驻：xx 段
- 摘要化：xx 段
- 迁出按需加载：xx 段
- 删除重复：xx 段
- 合并重写：xx 段
```

### 8.2 迁移映射表

```md
| 原位置 | 原内容 | 动作 | 新位置 |
|---|---|---|---|
| AGENTS 第四章 | 长路径索引 | SUM | rules/*/index.md |
| Developer 工具段 | Explore 示例 | MOVE | agent guide |
```

### 8.3 风险检查表

```md
- [ ] 非实现请求仍能稳定不进入实现
- [ ] 范围不清时仍会优先澄清
- [ ] 不会擅自 commit/push
- [ ] 外部库仍会优先 librarian
- [ ] 多模块仍会优先 explore
- [ ] 关键红线没有被删掉
```

---

## 九、使用提醒

- 不要只删不迁；下沉内容必须有新入口承接。
- 不要让索引页重新长成正文。
- 不要让同一规则在多个层级继续保留主定义。
