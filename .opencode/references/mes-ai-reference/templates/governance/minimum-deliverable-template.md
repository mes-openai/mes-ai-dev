# 最小可交付模板

> 本模板用于记录当前阶段或任务已形成的“最小可继续结果”。
> 最小可交付不等于完整完成，它用于说明：在未达到完整交付状态时，为什么当前结果已经足以支撑下一步继续推进。
> 本模板主要用于 GSD 模式，也可在 Strict 模式下用于中间检查点说明。

---

## 一、基本信息

| 字段 | 内容 |
|------|------|
| 需求编号 | REQ-YYYYMMDD-XXX / EMG-YYYYMMDD-XXX / N/A |
| 阶段 | analyze / design / develop / test / deliver / refresh / emergency |
| 当前模式 | strict / gsd |
| 当前目标 | [当前目标] |
| 最小可交付名称 | [名称] |
| 记录时间 | YYYY-MM-DD HH:mm:ss |
| 生成人/Agent | [名称] |
