# 代码生成真实性专项回归检查清单

> 本清单用于回归验证“数据库设计瞎写、import 错、MyBatis namespace 错、调用不存在方法、接错返回值”这 5 类真实问题是否已被骨架规则、skill、模板与门禁联合阻断。

## 一、数据库设计真实性

- [ ] 设计文档中的每张表是否都标注了“现有表 / 修改表 / 新增表”
- [ ] 每张表是否都填写了事实来源
- [ ] 每张表是否都填写了证据路径
- [ ] 若为新增表，是否填写了新增批准依据
- [ ] 是否仍存在把需求中的业务概念直接写成真实表的情况

## 二、Java 引用可解析性

- [ ] 生成代码中的包声明是否与目录路径一致
- [ ] import 对应类是否真实存在
- [ ] 异常类、分页对象、Converter、响应包装是否都来自真实项目代码
- [ ] 是否仍存在直接照搬模板默认依赖名的情况

## 三、MyBatis 映射一致性

- [ ] XML `namespace` 是否等于 Mapper 接口全限定名
- [ ] XML `id` 是否与接口方法一一对应
- [ ] `parameterType` / `resultType` / `resultMap` 是否都可解析到真实对象
- [ ] 表名、字段名、Schema 是否有真实 DDL / database-index / 现有 SQL 证据

## 四、方法存在性

- [ ] DTO / VO / Entity 上被调用的方法是否真实存在
- [ ] Mapper / Converter / helper / 上下文工具方法是否真实存在
- [ ] 是否仍存在模板示例方法被直接落代码的情况（如 `validateCreateParam`、`getCurrentUser`）

## 五、Provider 契约一致性

- [ ] 调用前是否读取 provider 真实接口定义、client 封装或 API 注册信息
- [ ] 调用参数类型是否与 provider 真实签名一致
- [ ] 返回值类型、泛型、包装层是否一致
- [ ] 调用方解包方式是否与 provider 真实返回结构一致

## 五点五、图谱辅助真实性

- [ ] GitNexus 类图谱结果是否只作为候选线索，未替代真实代码定义、API 注册信息、依赖图或测试证据
- [ ] graphify 类导读是否只作为关系说明，未替代正式设计、任务计划、测试报告或交付结论
- [ ] 图谱推断是否标注为候选/待复核，未写成确认事实
- [ ] 图谱发现的非目标问题是否未被混入当前代码生成范围

## 五点六、单元测试真实性

- [ ] 生成的单元测试是否兼容 Windows / Linux 路径差异
- [ ] 是否不存在硬编码盘符、用户目录、工作目录、`\\` 或 `/` 路径分隔符
- [ ] Mockito matcher 是否未与裸值混用导致参数 mismatch
- [ ] 复杂参数是否使用 `ArgumentCaptor` 或 `argThat(...)` 做精确校验
- [ ] stub 返回链是否完整，未留下空指针式 mock 缺口
- [ ] 是否未通过万能 `any()`、删除断言、吞异常或弱断言制造虚假通过

## 六、结论

- [ ] 5 类问题均已被规则 / skill / 模板 / 门禁联合覆盖，可进入下一阶段
- [ ] 有条件通过，仍需补充部分阻断能力
- [ ] 不通过，需回流骨架整改
