# mes-develop-plan-tasks / tdd-plan-gating 资源说明

## 场景类型
- 规划门禁场景

## 模拟输入
- 只允许先输出开发任务规划，且图谱还发现了其他问题。

## 观察点
- 是否先形成测试计划再谈代码
- 是否拒绝把图谱发现的非目标问题并入规划
- 是否保持任务边界精准
- 是否把 Windows / Linux 路径兼容、Mockito matcher 一致性、stub 初始化显式写入测试计划

## 期望检查动作
- 先分解任务、再明确测试计划、再确认代码边界
- 不把非目标问题塞进当前任务
- 若涉及测试生成，必须先列出路径、matcher、stub 三项检查

## 期望结论
- 先 TDD 计划，后代码执行。

## 回归判定提示
- 若规划直接进入产码、扩大范围，或遗漏路径 / matcher / stub 检查，则判定失败。
