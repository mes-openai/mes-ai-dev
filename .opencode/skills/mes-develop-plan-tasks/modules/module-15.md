# 审核要点

## 审核要点

### 任务完整性
- [ ] 是否覆盖所有设计文档中的功能点
- [ ] 是否包含所有数据库变更
- [ ] 是否包含所有API实现
- [ ] 是否包含所有前端页面

### 依赖正确性
- [ ] 依赖关系是否正确（无循环依赖）
- [ ] 并行组划分是否合理
- [ ] 前置任务是否准确
- [ ] 若使用 GitNexus 类能力，最小改动集合、消费者与回归路径是否已映射到任务计划
- [ ] 若使用 graphify 类能力，任务关系导读是否未替代正式任务拆解

### 工时合理性
- [ ] 预估工时是否合理
- [ ] 是否考虑了复杂度
- [ ] 是否预留了联调时间

### 可执行性
- [ ] 任务粒度是否适中（单个任务不超过8小时）
- [ ] 任务描述是否清晰
- [ ] 是否能被开发者直接理解

### 风险识别
- [ ] 是否识别了技术风险
- [ ] 是否标注了不确定因素
- [ ] 是否提出了应对方案

### 精准修改审核
- [ ] 是否明确排除了无关格式化、无关重命名、无关重构和顺手优化
- [ ] 是否避免把图谱发现的非目标问题纳入当前任务
