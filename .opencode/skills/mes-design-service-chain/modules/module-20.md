# 审核要点

## 审核要点

设计完成后，需从以下维度进行自审：

### 完整性审核
- [ ] 是否覆盖了所有服务调用关系
- [ ] 是否明确了所有调用方式（同步/异步）
- [ ] 是否设计了数据流转路径
- [ ] 是否考虑了事务边界

### 一致性审核
- [ ] 调用链路是否与API设计一致
- [ ] 数据流转是否与服务依赖关系一致
- [ ] 异常处理是否与服务规范一致

### 可靠性审核
- [ ] 是否设计了重试和降级策略
- [ ] 是否考虑了服务雪崩风险
- [ ] 是否设计了熔断机制
- [ ] 是否考虑了数据一致性

### 性能审核
- [ ] 是否优化了调用链路
- [ ] 是否考虑了并行调用
- [ ] 是否考虑了缓存
- [ ] 是否评估了性能影响
