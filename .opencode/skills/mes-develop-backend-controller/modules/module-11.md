# 审核要点

## 审核要点

### API设计一致性
- [ ] HTTP路径是否与API设计一致
- [ ] HTTP方法是否与API设计一致
- [ ] 请求参数是否与API设计一致
- [ ] 响应格式是否与API设计一致

### RESTful规范
- [ ] 是否遵循RESTful设计原则
- [ ] 路径命名是否规范
- [ ] HTTP方法使用是否正确
- [ ] 资源ID是否使用路径参数

### 参数校验
- [ ] @RequestBody参数是否有@Valid
- [ ] @PathVariable参数是否有校验注解
- [ ] @RequestParam参数是否有校验注解
- [ ] 校验注解是否正确

### 响应格式
- [ ] 是否使用统一Result封装
- [ ] 成功/失败响应是否正确
- [ ] HTTP状态码是否合理
- [ ] 响应数据结构是否正确

### 异常处理
- [ ] 是否捕获BusinessException
- [ ] 是否捕获系统异常
- [ ] 异常信息是否友好
- [ ] 是否有全局异常处理器

### 日志规范
- [ ] 是否记录请求日志
- [ ] 是否记录异常日志
- [ ] 日志级别是否合理
- [ ] 是否避免敏感信息泄露

### 安全考虑
- [ ] 是否有权限校验（如需要）
- [ ] 是否有数据权限校验（如需要）
- [ ] 是否防止参数篡改
- [ ] 是否防止接口滥用
