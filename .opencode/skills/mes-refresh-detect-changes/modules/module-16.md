# 注意事项

## 注意事项

1. **首次同步**：若无同步记录，执行全量扫描
2. **Git历史**：依赖Git历史，确保仓库状态正常
3. **文件分类**：准确识别文件类型，影响后续刷新流程
4. **增量更新**：只处理变更部分，避免全量重建
5. **先写主文件摘要并默认双写明细**：同步状态先写入 `state.yaml.sync` 摘要字段，再同步写 `state-detail/sync.yaml`
6. **不默认要求 sync 明细留在主文件**：`last_commit` 与 `changes` 等字段可在双写期迁入 `state-detail/sync.yaml`
7. **兼容视图联合渲染**：`.sync-record.json` 的渲染口径应与双写方案一致，不能再隐含要求主文件保留全部明细
