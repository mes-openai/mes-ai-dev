# 执行步骤（1/2）

## 执行步骤

### 1. 输出执行计划

```
本次执行计划：
目标：按部署计划执行部署操作
步骤：
  1. 读取部署计划，确认部署步骤
  2. 确认验收已通过
  3. 等待人工确认
  4. 执行前置检查
  5. 按序执行部署步骤
  6. 记录每步执行结果
  7. 执行部署后验证
  8. 生成部署日志
预期产出：deploy-log.md
风险：部署步骤执行失败、回滚不及时
```

### 2. 前置条件确认

1. 读取验收报告，确认验收结论为"通过"或"有条件通过"
2. 如验收未通过 → **终止部署**，提示先解决验收问题
3. 读取部署计划，确认部署步骤清单
4. 检查前置条件：
   - 目标环境是否就绪
   - 部署包是否已构建
   - 数据库脚本是否准备就绪

**Step Gate A**：前置条件确认不完整或验收结论不满足部署要求 → 打回步骤2重做，不得进入人工确认。

### 3. 人工确认（强制门禁）

**本步骤不可跳过。** 必须等待用户明确回复"确认部署"后才能继续。

输出确认信息：
```
⚠️ 部署确认

需求编号：REQ-YYYYMMDD-XXX
部署环境：[环境名称]
部署策略：[蓝绿/滚动/灰度]
涉及服务：[服务列表]
回滚方案：[简要描述]

请确认是否执行部署？
```

### 4. 执行前置检查

1. 检查目标环境连通性
2. 检查磁盘空间是否充足
3. 检查当前服务健康状态
4. 备份当前版本信息（用于回滚）

**Step Gate B**：人工确认缺失或前置检查未通过 → 打回步骤3-4重做，不得进入按序部署。

### 5. 按序执行部署

按部署计划中的顺序，逐步执行：

| 步骤类型 | 操作 | 验证方式 |
|---------|------|---------|
| 数据库变更 | 执行DDL/DML脚本 | 检查表结构/数据是否正确 |
| 配置变更 | 更新配置文件/配置中心 | 验证配置项值 |
| 后端部署 | 部署服务包 | 健康检查接口返回200 |
| 前端部署 | 部署静态资源 | 页面可正常访问 |
| 缓存刷新 | 清除/预热缓存 | 缓存命中率正常 |

每步执行后立即记录：
- 执行时间
- 执行结果（成功/失败）
- 耗时
- 异常信息（如有）

### 6. 部署后验证

1. 执行健康检查（所有服务的/health端点）
2. 执行冒烟测试（核心业务流程验证）
3. 检查监控指标是否正常
4. 检查日志是否有异常错误

**Step Gate C**：部署执行记录不完整或部署后验证未通过 → 打回步骤5-7重做，不得生成部署日志。

### 7. 回滚处理（仅在失败时）

如任何步骤执行失败或部署后验证不通过：

1. 立即停止后续部署步骤
2. 按回滚方案逆序执行回滚
3. 回滚后再次执行健康检查
4. 记录回滚原因和过程
5. 输出失败报告供人工排查
