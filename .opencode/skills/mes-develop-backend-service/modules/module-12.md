# 审核要点

## 审核要点

### 业务逻辑正确性
- [ ] 是否正确实现设计文档中的业务规则
- [ ] 参数校验是否完整
- [ ] 业务校验是否合理
- [ ] 异常处理是否正确

### 数据转换正确性
- [ ] DTO转Entity是否正确
- [ ] Entity转VO是否正确
- [ ] 批量转换是否正确
- [ ] 空值处理是否合理

### 事务管理
- [ ] 写操作是否加了@Transactional
- [ ] rollbackFor是否配置正确
- [ ] 事务边界是否合理
- [ ] 是否有嵌套事务问题

### 异常处理
- [ ] 是否捕获所有可能的异常
- [ ] 是否使用统一的异常类型
- [ ] 异常信息是否清晰
- [ ] 是否记录异常日志

### 日志规范
- [ ] 是否记录关键操作日志
- [ ] 日志级别是否合理
- [ ] 日志信息是否完整
- [ ] 是否避免敏感信息泄露

### 跨服务调用
- [ ] 调用方式是否正确
- [ ] 服务名是否从配置获取
- [ ] 异常处理是否完整
- [ ] 是否考虑超时和重试
- [ ] provider 真实返回值、泛型和包装层是否已核对
- [ ] 调用方接收返回值的类型与解包方式是否正确

### 性能考虑
- [ ] 是否避免N+1查询
- [ ] 批量操作是否合理
- [ ] 是否有不必要的数据库查询
- [ ] 分页查询是否高效
