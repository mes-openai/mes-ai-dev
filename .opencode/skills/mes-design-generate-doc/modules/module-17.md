# 约束规则

## 约束规则

1. **必须先输出执行计划** - 在任何文档整合前，必须明确目标、步骤、预期产出
2. **内容完整性** - 必须整合所有设计阶段的产出，不得遗漏
3. **一致性检查** - 必须检查不同设计文档间的一致性，消除冲突
4. **模板规范** - 必须按照已确认的设计文档模板结构组织内容；模板来源优先级为用户显式提供模板、项目级自定义模板 `.opencode/references/mes-ai-reference/templates/design/custom-design-doc-template.md`、默认模板 `.opencode/references/mes-ai-reference/templates/design/design-doc-template.md`
5. **版本管理** - 必须标注文档版本和变更历史
6. **易读性** - 文档必须清晰、规范、易于理解
7. **无冗余** - 避免内容重复，合并相同内容
8. **目录隔离** - 设计阶段输出目录必须是 `mes-ai-dev/workspace/designs/{REQ-ID}/`，不得沿用需求分析目录
9. **用户模板保真** - 用户模板中的固定章节、标题和顺序应优先保留；骨架必要内容缺失时以补充章节或附录方式补齐，不得擅自删除用户固定章节
10. **模板异常处理** - 用户提供模板路径不存在时必须暂停确认；项目级自定义模板不存在时可回退默认模板；用户以内联内容提供模板时必须在 `template-source.md` 留痕
