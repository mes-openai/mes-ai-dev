# 审核要点

## 审核要点

文档生成完成后，需从以下维度进行自审：

### 完整性审核
- [ ] 是否整合了所有设计阶段的产出
- [ ] 是否包含了所有必要的章节
- [ ] 是否有遗漏的设计内容
- [ ] 是否提供了完整的参考资料

### 一致性审核
- [ ] 不同章节间是否有内容冲突
- [ ] API与数据库定义是否一致
- [ ] 前端与API定义是否一致
- [ ] 服务调用与API定义是否一致

### 规范性审核
- [ ] 是否按照模板结构组织
- [ ] 是否声明并使用了正确的模板来源：用户模板 / 项目级自定义模板 / 默认模板
- [ ] 用户模板中的固定章节是否完整保留
- [ ] 用户模板路径不存在时是否已暂停确认，而非静默回退
- [ ] 使用用户内联模板或项目级自定义模板时是否生成 `template-source.md`
- [ ] 项目级自定义模板不存在且回退默认模板时是否在执行计划中说明
- [ ] 设计产物是否写入 `mes-ai-dev/workspace/designs/{REQ-ID}/`，未写入 `workspace/requirements/{REQ-ID}/`
- [ ] `{REQ-ID}` 是否按 US 优先、ITRQ 次之、兜底 REQ 编号的规则解析
- [ ] 格式是否统一规范
- [ ] 表格、代码块是否正确
- [ ] 版本信息是否完整

### 易读性审核
- [ ] 目录是否清晰
- [ ] 章节编号是否合理
- [ ] 内容是否易于理解
- [ ] 是否有足够的示例和说明
