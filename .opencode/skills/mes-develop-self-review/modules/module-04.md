# 执行步骤（2/2）

### 4. 六维审查

#### 4.1 代码规范审查
- 命名规范：类名、方法名、变量名、统一命名风格、避免无意义名称、避免拼音命名
- 注释规范：文件头注释、类/方法注释、必要字段注释、注释准确性
- 代码格式：缩进、空格、行长度、大括号位置
- 代码结构：重复代码、过长方法、过度嵌套、合理分层

#### 4.2 逻辑正确性审查
- 数据库逻辑：SQL 语法、表结构与设计一致性、索引合理性、约束正确性
- 后端逻辑：字段映射、业务逻辑、事务管理、异常处理
- 前端逻辑：数据流、状态管理、事件处理、组件交互
- 接口对接：API 路径、请求参数、响应格式、错误处理

#### 4.3 一致性审查
- 与设计文档一致性：数据库设计、API 设计、业务逻辑、UI 设计
- 与现有代码一致性：命名风格、代码结构、注释风格、配置方式
- 与存量项目结构一致性：目录位置、分层模式、依赖组织、测试组织、参考实现模式
- 代码模式一致性：
  - 使用 `templates/design/code-pattern-consistency-template.md` 记录参考实现对齐情况
  - 产出写入：`mes-ai-dev/workspace/development/REQ-YYYYMMDD-XXX/code-pattern-consistency.md`
- 前后端一致性：数据结构、字段名、类型、格式

#### 4.4 完整性审查
- 功能完整性：功能点、交互、状态是否遗漏
- 数据完整性：字段、索引、约束、初始化数据是否完整
- 接口完整性：接口、参数、响应、错误码是否完整
- 配置完整性：必要配置、环境变量、回滚配置、日志配置

#### 4.5 安全性审查
- SQL 安全：参数化查询、SQL 注入防护、权限控制、敏感数据加密
- 接口安全：权限校验、参数校验、数据权限、接口滥用防护
- 前端安全：XSS、CSRF、敏感数据保护、权限控制
- 配置安全：敏感信息硬编码、明文密码 / 密钥、加密方案

#### 4.6 性能审查
- 数据库性能：索引、全表扫描、N+1 查询、批量操作
- 后端性能：不必要查询、循环、缓存、异步处理
- 前端性能：不必要渲染、组件拆分、懒加载、资源优化

**Step Gate C**：六维审查有任一维度未完成、问题分级不清或证据不足 → 打回步骤4重做，不得进入设计偏差记录。

### 5. 设计偏差记录

> 审查代码与设计文档的一致性时，发现设计不完整、不合理、或与代码实际实现不符的情况，必须记录。

#### 偏差处理规则
1. **设计不完整**：在代码中按最佳实践补充实现，记录偏差，标注需补充设计文档
2. **设计不合理**：在代码中采用合理方案实现，记录偏差，提交变更申请（使用 `templates/governance/change-management-template.md`）
3. **设计与代码不符（开发者偏差）**：以设计为准修改代码，除非有充分理由偏离
4. **设计与代码不符（设计过时）**：记录偏差，提交变更申请更新设计文档
5. **所有偏差** 必须在自审报告中列出，并在“设计偏差”章节汇总
6. **涉及变更申请的偏差**：引用变更编号，跟踪变更审批状态

**Step Gate D**：设计偏差未记录、变更申请关联缺失或偏差判断不清 → 打回步骤5重做，不得生成最终自审报告。

### 6. 使用模板生成报告
创建文件：`mes-ai-dev/workspace/development/REQ-YYYYMMDD-XXX/self-review-report.md`

统一使用：
- `mes-ai-dev/templates/governance/self-review-report-template.md`
- `mes-ai-dev/templates/governance/design-deviation-template.md`
- `mes-ai-dev/templates/governance/detailed-review-report-template.md`（落盘模板）
- `mes-ai-dev/knowledge/rules/governance/review-report-standard.md`（统一最小字段集约束）

> **模板使用要求**：
> - 自审报告模板承接六维审核结果、问题分级、评分与结论
> - 设计偏差模板承接偏差明细、统计与变更申请关联
> - 不得保留模板占位文本，必须替换为真实审查结果
> - 自审报告必须补齐风险评估、证据路径、复审/闭环状态等详细审查报告字段
> - 如主报告超限，必须拆分附录，不得压缩掉高优问题、关键风险和整改要求
> - 报告中必须单列：TDD 用例计划确认情况、新生成测试用例全绿结论、覆盖率 100% 结论、存量结构贴合结论

**Step Gate E**：自审报告/偏差记录缺少真实结论、评分依据、风险评估、证据路径或闭环状态 → 打回步骤6重做，不得交付下游阶段。
