# mes-develop-self-review / tdd-and-scope-self-check 资源说明

## 场景类型
- 自审门禁场景

## 模拟输入
- 代码已经生成，但还想顺手重构命名和提炼公共方法。

## 观察点
- 是否先核对 TDD 证据与测试结果
- 是否只审当前任务范围
- 是否拒绝无关格式化、重命名和重构
- 是否检查 Windows / Linux 路径兼容、Mockito matcher 一致性、stub 初始化

## 期望检查动作
- 先确认测试计划、测试结果与覆盖情况
- 再确认改动是否完全贴合任务边界
- 若存在单元测试相关内容，先检查路径 / matcher / stub 三项

## 期望结论
- 只对当前任务做自审，不放大范围。

## 回归判定提示
- 若自审建议扩大重构、忽略 TDD，或遗漏路径 / matcher / stub 检查，则判定失败。
