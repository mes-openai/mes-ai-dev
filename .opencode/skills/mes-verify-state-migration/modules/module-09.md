# 约束规则

## 约束规则

### 必须遵守
1. **只读验证** — 不修改任何状态文件，仅读取检查
2. **只输出报告** — 仅写入报告文件，不修改知识库
3. **grep 检测必须检查上下文** — 命中关键词必须检查是否为允许语境（兼容/渲染/历史遗留）
4. **值对比必须遵循字段映射表** — 按 `state-rendering-spec.md` 映射表对比
5. **冲突以 state.yaml 为准** — 不一致时标注摘要/兼容视图需修正
6. **报告必须包含日期** — 便于多次执行对比
7. **详细审查报告强制** — 输出必须符合详细审查报告统一标准，不得退化为摘要式结论

### 禁止行为
1. ❌ 不修改 state.yaml 或任何摘要/兼容视图文件
2. ❌ 不跳过 grep 残留检查
3. ❌ 不跳过值对比检查
4. ❌ 不以摘要/兼容视图值为准判断一致性
5. ❌ 不美化校验结果，残留如实记录
