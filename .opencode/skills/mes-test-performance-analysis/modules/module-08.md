# 约束规则

## 约束规则

### 执行前必须输出计划
每次执行前必须输出执行计划，经用户确认后才可继续。

### 全面覆盖
必须覆盖所有性能检查维度：
- N+1查询
- 大事务
- 慢SQL
- 缺少索引
- 数据量控制

### 精准定位
问题报告必须包含：
- 文件路径
- 具体行号（或代码片段）
- 影响程度
- 问题描述
- 具体优化方案

### 影响分级
按照高/中/低三级划分影响程度：
- **高**：可能导致系统卡顿、超时
- **中**：可能影响部分场景性能
- **低**：优化后性能提升有限

### 提供可执行方案
优化建议必须是可执行的具体方案：
- SQL语句示例
- 索引创建语句
- 代码修改示例
