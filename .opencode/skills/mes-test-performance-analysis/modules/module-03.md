# 执行步骤（1/2）

## 执行步骤

### 1. 输出执行计划
在执行前，必须输出本次计划：
```
本次执行计划：
目标：对代码变更进行性能分析
步骤：
  1. 读取Service层代码，检查N+1查询
  2. 读取DAO层代码，检查慢SQL
  3. 读取数据库设计，检查索引缺失
  4. 分析事务边界，检查大事务
  5. 生成性能分析报告
预期产出：性能分析报告
风险：遗漏性能瓶颈、误报风险
```

### 2. 读取Service层代码
读取所有Service实现类文件：
- `jalor/service-xxx/src/main/java/com/jalor/xxx/service/impl/XxxServiceImpl.java`

**检查项**：
1. **N+1查询**：
   - 循环中调用DAO方法（每条数据触发一次查询）
   - 嵌套对象懒加载导致的多次查询
   - 批量操作中未使用批量查询方法

2. **大事务**：
   - @Transactional方法中包含远程调用
   - 事务范围过大（包含多个不相关的操作）
   - 长事务导致锁等待

3. **缺少分页的列表查询**：
   - 无LIMIT的查询
   - 未设置分页参数的列表返回
   - 全量返回大量数据

4. **不必要的全表扫描**：
   - 查询条件未使用索引字段
   - LIKE '%xxx%' 开头模糊查询
   - OR条件导致索引失效

**Step Gate A**：Service/DAO/数据库输入读取不完整或性能检查范围不清 → 打回步骤2-4重做，不得进入事务边界分析。

### 3. 读取DAO层代码
读取Mapper XML文件：
- `jalor/service-xxx/src/main/resources/mapper/XxxMapper.xml`

**检查项**：
1. **慢SQL**：
   - 查询条件字段是否缺少索引
   - 多表关联是否使用索引
   - 排序字段是否有索引

2. **大量数据查询**：
   - 无LIMIT约束的查询
   - COUNT(*)全表统计
   - 未分页的大数据查询

3. **笛卡尔积**：
   - 多表关联缺少WHERE条件
   - JOIN条件不完整
   - 子查询返回大量数据

### 4. 读取数据库设计
读取数据库设计文档和DDL脚本：
- `mes-ai-dev/workspace/designs/REQ-YYYYMMDD-XXX/database-design.md`
- `mes-ai-dev/workspace/development/REQ-YYYYMMDD-XXX/database/ddl-xxx.sql`

**检查项**：
1. **缺少索引的高频查询字段**：
   - WHERE条件字段是否有索引
   - JOIN关联字段是否有索引
   - ORDER BY排序字段是否有索引

2. **过大的字段类型**：
   - VARCHAR长度是否合理
   - TEXT类型是否必要
   - 大字段是否单独存储

### 5. 分析事务边界
检查事务注解使用：

**检查项**：
1. **事务范围过大**：
   - 一个事务包含多个不相关的操作
   - 事务方法嵌套调用其他事务方法
   - 事务中包含耗时的非数据库操作

2. **远程调用在事务中**：
   - 事务方法调用外部服务
   - 事务中发送HTTP请求
   - 事务中调用消息队列
