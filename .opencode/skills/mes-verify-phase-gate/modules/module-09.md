# 硬性约束

## 硬性约束

- **不可跳过**：每个 Command 必须在开始和结束时调用本 Skill
- **步骤级也不可跳过**：每个步骤输出进入下一步骤前必须调用本 Skill 或等价步骤级门禁
- **must-pass 不可降级**：must-pass 项的 ❌ 不能降级为 ⚠️
- **人工确认不可自动填充**：需要人工确认的项，必须等待真实的人工回复
- **状态门禁不可脱离规范解释**：凡涉及 `state.yaml` / 摘要或兼容视图的门禁项，必须按 `state-rendering-spec.md` 判断主从关系与一致性
- **收敛门禁必须读取收敛状态**：凡阶段依赖全仓视角结果，不得跳过 `state.yaml.initialization.convergence` 校验
- **未通过结果不得下游消费**：任一步骤/阶段 gate 为 ❌ 或未完成闭环的 ⚠️，不得注入下一步骤
- **门禁留痕必须详细**：需要落盘的 gate 结果，必须形成详细审查报告，不得以摘要替代
- **TDD 前置不可跳过**：命中 AI 代码生成时，未完成测试计划、用户补充与用户确认，不得放行代码生成
- **覆盖率不足只能补测**：不得通过删除、降级或跳过已通过测试用例换取覆盖率达标
- **OpenSpec 交接命名不可漂移**：阶段主交接文档必须使用 `spec.md`、`design.md`、`tasks.md`、`test-report.md`、`handover-doc.md` 等 OpenSpec 命名
