# 约束规则

## 约束规则

### 执行前必须输出计划
每次执行前必须输出执行计划，经用户确认后才可继续。

### 必须读取现有风格
在编写任何类前，必须：
1. 读取目标服务现有的Entity/DTO/VO类（至少各1个）
2. 分析注解使用、字段命名、代码风格
3. 保持与现有代码风格一致

### 字段映射规则
- Entity字段与数据库字段一一对应
- DTO字段根据API入参定义
- VO字段根据API出参定义
- 字段类型必须匹配数据库类型

### 命名规范
- Entity类名：`XxxEntity`（表名驼峰+Entity）
- DTO类名：`XxxDTO`（功能名+DTO）
- VO类名：`XxxVO`（功能名+VO）
- 字段名：驼峰命名，与数据库字段对应

### 注释要求
- 类必须有类注释（说明、作者、日期）
- 所有字段必须有注释
- 方法必须有注释（转换器类）

### 依赖约束
- Entity使用JPA注解
- DTO使用Validation注解
- VO使用Jackson注解
- 统一使用Lombok简化代码

### 不硬编码服务名
在代码中引用其他服务时，使用配置或常量，不硬编码服务名或URL。
