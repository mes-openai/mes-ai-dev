# 约束规则

## 约束规则

### 执行前必须输出计划
每次执行前必须输出执行计划，经用户确认后才可继续。

### 所有变更必须可回滚
每个DDL/DML操作必须有对应的回滚操作，确保迁移失败时可恢复。

### 迁移顺序必须合理
按照依赖关系设计顺序，避免因顺序错误导致迁移失败。

### 风险评估必须全面
评估所有可能的风险：
- 锁表风险
- 数据完整性风险
- 服务可用性风险
- 回滚风险

### 前置检查必须完整
迁移前必须检查：
- 磁盘空间是否充足
- 数据库是否已备份
- 是否有未释放的锁
- 是否有正在运行的大事务

### 数据校验必须设计
必须有数据完整性校验方案，确保迁移后数据正确。
