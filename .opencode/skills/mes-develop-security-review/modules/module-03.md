# 执行步骤（1/2）

## 执行步骤

### 1. 输出执行计划
在执行前，必须输出本次计划：
```
本次执行计划：
目标：对代码变更进行安全审查
步骤：
  1. 读取所有开发产出代码
  2. 检查SQL注入风险
  3. 检查XSS风险
  4. 检查越权访问风险
  5. 检查敏感数据泄露风险
  6. 检查接口鉴权
  7. 检查文件上传安全
  8. 检查CSRF防护
  9. 生成安全审查报告
预期产出：安全审查报告（security-review-report.md，详细审查报告）
风险：遗漏安全漏洞、误报风险
```

### 2. 读取开发产出代码
按顺序读取所有开发产出的代码文件：
1. Controller类：`jalor/service-xxx/src/main/java/com/jalor/xxx/controller/XxxController.java`
2. Service实现类：`jalor/service-xxx/src/main/java/com/jalor/xxx/service/impl/XxxServiceImpl.java`
3. Mapper XML文件：`jalor/service-xxx/src/main/resources/mapper/XxxMapper.xml`
4. Entity/DTO/VO类：对应的Model类文件

**Step Gate A**：安全审查输入范围不完整或关键代码文件缺失 → 打回步骤2重做，不得进入具体安全检查。

### 3. SQL注入检查
检查Mapper XML文件和Service层代码：

**检查项**：
- 是否使用`#{}`参数化查询（安全）
- 是否使用`${}`字符串拼接（高风险）
- 动态SQL中的条件是否正确转义
- LIKE查询是否使用CONCAT方式拼接

**风险等级**：
- **高**：使用`${}`拼接SQL
- **中**：动态SQL缺少条件校验
- **低**：LIKE查询未正确处理

### 4. XSS检查
检查Controller层和VO类：

**检查项**：
- Controller是否对输入参数进行校验和转义
- VO类是否对输出字段进行编码
- 是否存在直接返回用户输入的场景
- 富文本字段是否有专门的过滤处理

**风险等级**：
- **高**：直接返回未处理的用户输入
- **中**：缺少输入校验
- **低**：输出格式化不完整

### 5. 越权访问检查
检查Controller类和Service层：

**检查项**：
- API是否有权限校验注解（如@PreAuthorize）
- 数据操作是否校验数据归属（用户只能操作自己的数据）
- 查询接口是否有数据权限过滤
- 批量操作是否有权限校验

**风险等级**：
- **高**：无权限校验的敏感操作
- **中**：缺少数据归属校验
- **低**：查询权限不明确

### 6. 敏感数据泄露检查
检查日志输出和响应数据：

**检查项**：
- 日志是否打印敏感信息（密码、身份证、手机号等）
- 响应是否包含不必要的敏感字段
- 错误信息是否泄露系统内部细节
- 异常堆栈是否暴露给前端

**风险等级**：
- **高**：日志打印完整敏感数据
- **中**：响应包含敏感字段
- **低**：错误信息不够友好
