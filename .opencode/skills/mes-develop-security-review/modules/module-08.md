# 约束规则

## 约束规则

### 执行前必须输出计划
每次执行前必须输出执行计划，经用户确认后才可继续。

### 全面覆盖
必须覆盖所有安全检查维度，不遗漏任何检查项。

### 精准定位
问题报告必须包含：
- 文件路径
- 具体行号（或代码片段）
- 风险等级
- 问题描述
- 修复建议
- 证据路径与闭环状态

### 风险分级
按照高/中/低三级划分风险：
- **高**：可能导致数据泄露、系统被攻击
- **中**：可能被利用但影响有限
- **低**：存在隐患但难以利用

### 不误报
避免过度报告，只报告真实存在的安全问题。

### 详细审查报告强制
安全审查报告必须符合统一详细审查报告标准，不得退化为单表格或简略摘要。
