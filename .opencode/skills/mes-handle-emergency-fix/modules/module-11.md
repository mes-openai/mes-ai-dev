# 硬性约束

## 硬性约束

- **最小修改原则**：紧急修复只修问题，不做优化
- **事后必补**：24小时内必须补充完整文档
- **不可跳过验证**：即使是紧急修复，冒烟测试不可省略
- **必须记录**：所有紧急修复必须记录到 status-tracker.md
- **图谱只作辅助**：GitNexus 类结果只能用于定位故障入口、调用链、最小修复点和冒烟路径，不得替代故障事实或验证结果
- **导读不替代报告**：graphify 类事故关系导读不得替代事件报告、postmortem、回流动作或阶段门禁
- **禁止扩大范围**：不得因图谱发现非当前故障问题而引入无关重构、无关格式化或非止血目标变更
