#### 2. 快速定位

1. 读取相关服务的 `service-xxx/detail.md` 快速定位代码范围
2. 读取 `file-summaries.md` 找到具体文件
3. 直接读取相关源文件分析问题

**上下文预算放宽**：紧急修复允许跳过第0/1层，直接读取相关源文件

#### 3. 修复实施

1. 最小化修改 — 只修复问题本身，不做重构
2. 记录修改内容到 `incident-report.md`
3. 本地编译验证

#### 4. 快速验证

1. 冒烟测试（核心业务流程）
2. 回归测试（修改点相关的测试用例）
3. 不要求完整测试套件，但必须覆盖修改点

#### 5. 快速部署

1. 获取口头/书面审批
2. 按环境治理规则部署（DEV → TEST → PROD 可合并，但必须验证）
3. 部署后立即执行冒烟测试

#### 6. 事后补充

紧急修复完成后 **24 小时内** 必须补充：

| 补充项 | 说明 |
|--------|------|
| 完整的事故报告 | root cause + 修复方案 + 预防措施 |
| 回归测试 | 补充完整的回归测试用例 |
| 知识库更新 | 如修复涉及架构变更，执行 /mes-refresh-knowledge |
| 更新 status-tracker | 在需求总表记录此次紧急修复 |
