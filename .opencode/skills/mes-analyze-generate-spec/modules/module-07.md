# 约束规则

## 约束规则

### 必须遵守
1. **执行前输出计划**：在任何操作前必须输出执行计划
2. **结果审核**：生成文档前必须审核内容的完整性、准确性
3. **按模板组织**：严格按照标准模板结构组织内容
4. **迭代需求特殊处理**：若为迭代需求必须增加“存量功能变更说明”章节
5. **内容整合**：从已有分析文件整合内容，不重新分析
6. **文件交接**：通过已有文件获取输入，不依赖对话历史

### 禁止行为
1. ❌ 不重新进行需求分析（使用已有分析结果）
2. ❌ 不省略模板中的章节
3. ❌ 不猜测未分析的信息
4. ❌ 不遗漏迭代需求的特殊章节
5. ❌ 不修改已有分析的结论

### 文档生成策略
```
策略1：整合已有分析
  raw-requirement.md → 需求要素、影响分析、仓库清单
  business-flow-trace.md → 流程追踪、接口信息

策略2：按模板组织
  按模板章节顺序组织内容
  每章节从对应分析文件提取信息

策略3：迭代需求特殊处理
  检查需求类型
  若为迭代需求 → 增加"存量功能变更说明"章节
```
