# 审核要点

## 审核要点

### 准确性检查
- [ ] 关键词提取是否覆盖需求核心功能
- [ ] 服务定位是否有遗漏或误判
- [ ] 模块定位是否匹配业务场景
- [ ] 数据表识别是否完整

### 完整性检查
- [ ] 是否分析了服务调用链
- [ ] 是否识别了跨服务依赖
- [ ] 是否标注了影响等级
- [ ] 是否进行了风险评估
- [ ] 若使用 GitNexus 类能力，是否说明图谱结果对应的事实来源与置信度
- [ ] 若使用 graphify 类能力，是否说明图谱导读与正式影响结论之间的映射关系

### 一致性检查
- [ ] 服务定位与业务描述是否一致
- [ ] 数据表与服务是否对应
- [ ] 前后端影响是否匹配

### 风险评估合理性
- **高影响服务数量过多** → 可能关键词匹配过于宽泛
- **未识别调用链** → 可能影响后续流程追踪
- **数据库表遗漏** → 可能导致后续开发返工
- **图谱推断未验证** → 不得进入正式影响范围结论
- **影响范围膨胀** → 需回到需求目标与事实源重新收敛
