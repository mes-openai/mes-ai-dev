# 约束规则

## 约束规则

### 必须遵守
1. **执行前输出计划**：在任何操作前必须输出执行计划
2. **结果审核**：输出文件前必须审核解析结果的正确性
3. **不预设服务名**：不猜测涉及哪些微服务，留给 mes-analyze-impact-scope 分析
4. **不加载代码**：本阶段不读取任何代码文件
 - 但**必须读取**业务参考知识：`mes-ai-dev/knowledge/reference/terminology-glossary.md`（理解业务术语）和 `domain-model.md`（理解领域概念）
5. **保持中立**：忠实记录原始需求，不添加主观推断
6. **明确标注模糊点**：任何不清晰之处必须明确标注
7. **按需生成需求编号**：检查现有需求编号，避免冲突

### 禁止行为
1. ❌ 不读取代码文件（jalor/、web/）
2. ❌ 不猜测技术实现方案
3. ❌ 不修改原始需求描述的语义
4. ❌ 不跳过模糊点的识别
5. ❌ 不使用硬编码的服务名、模块名
