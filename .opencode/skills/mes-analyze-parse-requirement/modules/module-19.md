# 审核要点

## 审核要点

### 完整性检查
- [ ] 六大需求要素是否全部解析
- [ ] 模糊点是否全部识别
- [ ] 需求类型判断是否准确

### 一致性检查
- [ ] 需求描述内部是否存在矛盾
- [ ] 业务规则是否与功能描述冲突
- [ ] 权限矩阵是否与角色定义一致

### 可追溯性检查
- [ ] 需求编号是否唯一
- [ ] 迭代需求是否正确关联历史需求
- [ ] 待确认事项是否有责任人

### 风险评估
- **模糊点多** → 需求不明确，建议重新确认
- **业务规则缺失** → 后续开发可能返工
- **非功能需求未定义** → 可能影响系统质量
