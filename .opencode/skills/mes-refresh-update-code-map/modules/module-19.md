新增模块时，同时补充：

- `module-[模块名]/detail.md`：维护页面、API 调用、后端依赖等完整详情
- `module-[模块名]/file-summaries.md`：维护关键页面、组件、接口文件摘要

#### 修改模块处理

若模块有变更，按变更内容更新对应目录中的文件：

1. **新增页面**：更新 `detail.md` 中的页面路由表格，必要时同步更新 `index.md` 摘要
2. **新增API调用**：更新 `detail.md` 中的 API 调用表格，并在 `file-summaries.md` 增加对应文件摘要
3. **新增组件**：更新 `detail.md` 的组件说明，并补充 `file-summaries.md`
4. **修改结构**：更新 `index.md` 的精简索引，并在 `detail.md` 中补充目录结构说明

**Step Gate B**：后端/前端代码地图更新不完整、缺少摘要或误删索引 → 打回步骤2-3重做，不得进入总览与结果汇总。

#### 删除模块处理

若模块被删除，标记模块目录下相关文件为已删除：

```markdown
# module-[模块名]/index.md

> ⚠️ 此模块已删除
> 删除时间：[日期]
> 删除原因：[原因]
```
