# 约束规则

## 约束规则

### 执行前必须输出计划
每次执行前必须输出执行计划，经用户确认后才可继续。

### 必须读取现有风格
在编写任何页面前，必须：
1. 读取目标模块现有的页面文件（至少 3 个）
2. 分析页面结构、布局、路由配置、逻辑风格
3. 保持与现有代码风格一致

### 页面布局一致性
- 页面必须使用统一的布局结构
- 页面标题区域格式一致
- 页面内容区域格式一致
- 面包屑导航格式一致

### 路由配置规范
- 路由 path 使用小写 + 连字符：`/xxx-manage`
- 路由 name 使用驼峰：`XxxManage`
- meta 必须包含 title
- 需要权限的页面必须标记 requiresAuth

### 组件整合规范
- 页面组件引用业务组件
- 通过 props 传递数据
- 通过 events 处理交互
- 使用 ref 调用子组件方法

### 状态管理规范（如使用 Vuex）
- 共享状态使用 Vuex 管理
- 页面局部状态使用组件 data / ref / reactive
- 状态变更通过 mutations / actions 或既有 store 约定
- 不直接修改 state

### 路由守卫规范
- 需要权限检查的路由使用 beforeRouteEnter 或现有守卫机制
- 需要离开确认的页面使用 beforeRouteLeave 或等价机制
- 路由守卫逻辑保持清晰简单

### 不硬编码路径
路由路径使用配置或常量，不硬编码具体路径（除了路由配置中的 path）。
