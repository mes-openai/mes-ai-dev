# 审核要点

## 审核要点

### 页面结构正确
- [ ] 页面布局是否与现有页面一致
- [ ] 页面标题区域是否正确
- [ ] 页面内容区域是否正确
- [ ] 面包屑导航是否正确

### 组件整合正确
- [ ] 业务组件是否正确引用
- [ ] props 传递是否完整
- [ ] events 监听是否完整
- [ ] ref 使用是否正确

### 路由配置正确
- [ ] 路由 path 是否规范
- [ ] 路由 name 是否规范
- [ ] meta 信息是否完整
- [ ] 是否有路由冲突

### 页面逻辑正确
- [ ] 数据初始化是否正确
- [ ] 新增/编辑逻辑是否正确
- [ ] 删除逻辑是否正确
- [ ] 异常处理是否正确

### 样式一致性
- [ ] 页面样式是否与现有页面一致
- [ ] 布局样式是否一致
- [ ] 是否遵循样式变量
- [ ] 是否使用 scoped

### 功能完整性
- [ ] 是否实现设计文档所有功能
- [ ] 是否有遗漏的交互
- [ ] 是否有遗漏的状态
- [ ] 是否有遗漏的校验

### 用户体验
- [ ] 操作是否流畅
- [ ] 提示是否友好
- [ ] 错误处理是否合理
- [ ] 加载状态是否清晰

### 代码质量
- [ ] 是否有冗余代码
- [ ] 是否有重复逻辑
- [ ] 是否有潜在 bug
- [ ] 是否有性能问题
