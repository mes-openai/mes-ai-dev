# 执行步骤（1/2）

## 执行步骤

### 1. 输出执行计划
```
本次执行计划：
目标：提取实现模式目录，识别反模式，索引测试资产和运行时配置
步骤：
  1. 读取 patterns.md 模板结构
  2. 按服务分组读取代表性代码文件
  3. 提取后端实现模式（Controller/Service/DAO/VO-DTO/错误处理/权限/日志）
  4. 提取前端实现模式（页面/API调用/状态管理/组件）
  5. 识别反模式和遗留债务
  6. 扫描测试资产
  7. 扫描运行时配置
  8. 生成 patterns.md, test-assets.md, runtime.md
预期产出：3个索引文件
风险：模式提取可能有遗漏，需人工确认
```

### 2. 读取前置输入
- 各服务的 `file-summaries.md` — 确定要读取的代表性文件
- 每个服务读 2-3 个 Controller、1-2 个 Service、1-2 个 DAO

**Step Gate A**：前置输入读取不完整或代表性文件选择范围错误 → 打回步骤2重做，不得进入模式提取。

### 3. 提取实现模式
按领域分类提取典型写法：
- **Controller**：REST注解风格、响应包装、参数校验
- **Service**：事务管理、异步编排、业务校验
- **DAO/MyBatis**：CRUD模式、动态SQL、批量操作
- **VO/DTO**：命名规则、使用场景
- **错误处理**：异常类体系、错误码使用
- **权限校验**：鉴权方式、权限粒度
- **日志埋点**：日志级别、关键操作记录
- **公共契约模式**：统一响应包装、SDK 请求/响应模型、MQ/认证/网关对象的真实定义点与使用范围

### 4. 识别反模式
- 命名不一致区域（同一概念不同命名）
- 同一能力的多套实现
- 已废弃但仍被引用的模块
- 配置散落区
