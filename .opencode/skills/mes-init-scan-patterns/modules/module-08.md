# 约束规则

## 约束规则

### 必须遵守
1. **执行前输出计划** — 在任何操作前必须输出执行计划
2. **上下文管理** — 每服务最多读 5 个文件，调用 `mes-guard-context-budget`
3. **标注推荐度** — 每个模式标注 ★★★推荐 / ★★可选 / ★遗留
4. **区分推荐和遗留** — 明确标注"新代码应对齐"vs"遗留代码，不应模仿"

### 禁止行为
1. ❌ 不修改已有知识库文件
2. ❌ 不修改任何源码
3. ❌ 不跳过反模式识别
4. ❌ 不在未找到真实文件锚点时将抽象描述写成推荐模式
5. ❌ 发现多套返回包装或多套异常码风格时强行统一为单一模式
