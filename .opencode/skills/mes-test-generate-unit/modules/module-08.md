# 质量检查清单

## 质量检查清单

- [ ] 所有public方法都有测试覆盖
- [ ] 每个被测 public 方法对应一个 `@Nested` 内部类
- [ ] 每个方法至少有正常/异常两个场景
- [ ] 边界条件有测试覆盖
- [ ] 测试方法命名清晰
- [ ] 使用 JUnit 5 + Mockito + JDK8，未使用 JUnit 4 默认写法
- [ ] Mock设置正确
- [ ] 已 Mock 外部 API 调用、DAO 数据操作层、`I18nUtils` / `I18nUtil`、`GuidUtil` 等不可控依赖
- [ ] 断言完整且准确
- [ ] 返回对象测试同时断言状态码/结果标识和核心业务数据，未只断言状态码
- [ ] 行覆盖率、分支覆盖率、方法覆盖率均达到 100%
- [ ] 无冗余测试代码
- [ ] 符合现有测试风格
- [ ] 未测试 private 方法
- [ ] 未为了测试通过而修改测试逻辑或放宽断言
- [ ] 未使用 import 通配符
- [ ] 路径拼接和路径断言兼容 Windows / Linux，未硬编码盘符、工作目录或路径分隔符
- [ ] Mockito matcher 使用一致，未混用裸值与 matcher 导致参数 mismatch
- [ ] 复杂参数已使用 `ArgumentCaptor` 或 `argThat(...)` 做精确校验
- [ ] stub 返回链完整，未留下空指针式 mock 缺口
- [ ] 未通过万能 `any()`、删除断言或吞异常制造虚假通过
