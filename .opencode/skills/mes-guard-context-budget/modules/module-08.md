# 异常处理

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 单个服务 detail.md 超过 10K token | 标记为"超大服务"，只读取相关包路径的章节 |
| 涉及服务数>10 | 强制拆分为多批次，每批≤3个服务 |
| 源文件数>30 | 强制分批，每批≤15个文件 |
| 总预算>190K | 拒绝执行，必须先拆分需求或缩减范围 |
| 仓规模为`mega`但无hot层文件 | 阻断并提示先执行 `/mes-init-enrich` 深化hot层 |
| 仓规模为`large`/`mega`且需求涉及>5个服务 | 强制分批，按hot-services优先级排序处理顺序 |
