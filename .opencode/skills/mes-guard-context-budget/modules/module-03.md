# 上下文预算分配

## 上下文预算分配

### 单阶段预算（硬性上限）

| 预算项 | 分配 | 硬性上限 | 说明 |
|--------|------|---------|------|
| 系统提示 + AGENTS.md | 5K token | 5K | 固定 |
| Skill 提示词 | 3K token | 5K | 单个 Skill |
| 第0层总览 | 3K token | 5K | backend-overview + frontend-overview |
| 第1层精简版 | 10K token | 15K | 最多读取 5-8 个 index.md |
| 第1层完整版 | 15K token | 24K | 最多读取 2-3 个 detail.md |
| 第1.5层摘要 | 3K token | 5K | 最多读取 2-3 个 file-summaries.md |
| 第2层源码 | 25K token | 40K | 最多读取 10-15 个源文件 |
| 对话+中间产出 | 50K token | 100K | 视阶段复杂度 |
| **合计** | **~115K** | **~190K** | |

### 读取数量硬性限制

| 层级 | 单阶段最大读取量 | 超限处理 |
|------|----------------|---------|
| 第0层 | 2 个文件（backend + frontend overview） | 不可能超限 |
| 第1层精简版 | 8 个 index.md | 超过 → 只读与需求相关的，其余跳过 |
| 第1层完整版 | 3 个 detail.md | 超过 → 拆分为多批次，每批≤3个服务 |
| 第1.5层摘要 | 5 个 file-summaries.md | 超过 → 只读涉及修改的服务 |
| 第2层源码 | 15 个源文件 | 超过 → 分批处理，使用 workspace/ 交接 |
| 历史文档 | 5 个 workspace/ 下的文档 | 超过 → 只读当前阶段产出 + 上一阶段产出 |
