# 审核要点

## 审核要点

### 评审质量检查
- [ ] 是否完成五个维度的审核
- [ ] 是否基于知识库验证关联准确性
- [ ] 是否给出明确的问题优先级
- [ ] 是否给出明确的评审结论
- [ ] 是否提供修正建议

### 评审结论合理性
- **通过**：五个维度均无明显问题，或仅有低优先级问题
- **需修改后通过**：存在中优先级问题，修正后可通过
- **不通过**：存在高优先级问题，必须重新分析

### 问题分级标准
- **高优先级**：影响需求正确性、完整性、可行性的关键问题
- **中优先级**：影响需求质量、可测试性的重要问题
- **低优先级**：影响文档规范性、可读性的次要问题
