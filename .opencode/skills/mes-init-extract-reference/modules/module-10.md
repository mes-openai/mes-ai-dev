# 约束

## 约束
- 只读代码，不修改 `jalor/`、`web/` 与其他现有业务文件。
- 所有推断结果需标注置信度。
- 不存在的模式统一标注"未检测到"。
- 所有输出文件均使用中文。
- 结果写入前需执行正确性审核，确认逻辑一致、来源可追溯、结构完整。
- **存量仓约束**：优先提取真实边界（职责/API归属/表归属/外部集成），延后抽象术语体系。参考实现从代码中找到最规范模块，不凭空设定理想标准。所有推断标注置信度和推断依据。
- **术语表不可延后**：即使是大仓/超大仓，也必须在初始化/深化链路内产出可消费的术语表，不得延后到首次需求分析。
- **领域知识同类不可延后**：`domain-model.md`、`data-dictionary.md`、`enum-registry.md`、`error-code-registry.md`、`permission-matrix.md`、`api-conventions.md`、`coding-standards.md` 也必须形成可消费版本，不得长期停留在占位状态。
- **外部契约源优先级不可跳过**：不得跳过 workspace/repo/source artifact 直接反编译 jar。
- **无源不得产全局规范**：若来源类型为 `unknown`，不得将契约内容写成项目级确定事实。
