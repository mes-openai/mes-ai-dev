# 审核要点

## 审核要点

### API 路径正确性
- [ ] URL 是否与后端 Controller 一致
- [ ] HTTP 方法是否与 API 设计一致
- [ ] 资源 ID 是否使用路径参数
- [ ] 查询条件是否使用 params

### 函数命名规范
- [ ] 函数命名是否符合规范
- [ ] 是否与现有 API 函数风格一致
- [ ] 是否易于理解和使用

### 参数传递正确性
- [ ] POST/PUT 是否使用 data
- [ ] GET 是否使用 params
- [ ] DELETE 是否正确处理 ID
- [ ] 参数类型是否正确

### 注释完整性
- [ ] 是否有文件头注释
- [ ] 每个函数是否有注释
- [ ] 参数说明是否完整
- [ ] 返回值说明是否完整

### 类型定义正确性（TS）
- [ ] 接口定义是否完整
- [ ] 字段类型是否正确
- [ ] 是否有必填/可选标记
- [ ] 是否有字段注释

### 响应处理正确性
- [ ] 是否正确解构响应
- [ ] 是否处理成功/失败
- [ ] 是否处理异常
- [ ] 是否有错误提示

### 风格一致性
- [ ] 是否与现有 API 风格一致
- [ ] 请求封装是否一致
- [ ] 函数命名是否一致
- [ ] 注释格式是否一致
