# 约束规则

## 约束规则

1. **必须先输出执行计划** - 在任何设计活动开始前，必须明确目标、步骤、预期产出
2. **命名规范** - 表名使用`t_`前缀，索引名使用`idx_`前缀
3. **Schema识别** - 从数据库注册表动态获取Schema信息，不硬编码
4. **上下文管理** - 只加载涉及服务的Schema索引，不读取无关Schema
5. **数据安全** - 涉及删除字段或表时，必须明确标注数据丢失风险
6. **性能优先** - 所有索引设计需评估对性能的影响
7. **迁移可逆** - 数据迁移方案必须包含回滚步骤
8. **表真实性** - 每张表必须声明事实来源与证据路径；无证据不得写入设计文档
9. **新增表审批** - 新增表必须标注批准依据；没有批准依据不得按新增表输出
