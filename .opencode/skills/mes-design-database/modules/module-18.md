# 审核要点

## 审核要点

设计完成后，需从以下维度进行自审：

### 完整性审核
- [ ] 是否覆盖了所有数据实体
- [ ] 是否设计了必要的索引
- [ ] 是否提供了完整的DDL脚本
- [ ] 涉及数据迁移时是否设计了迁移方案
- [ ] Schema风险等级是否已标注（新增）
- [ ] 变更审批要求是否已确定（新增）
- [ ] 每张表是否都填写了真实性来源和证据路径

### 一致性审核
- [ ] 表结构是否与现有数据库风格一致
- [ ] 字段命名是否遵循现有规范
- [ ] 外键关系是否正确
- [ ] Schema归属是否正确
- [ ] 是否不存在把虚构表写成现有表的情况

### 性能审核
- [ ] 索引设计是否合理
- [ ] 是否评估了性能影响
- [ ] 是否考虑了查询优化
- [ ] 是否考虑了并发性能
- [ ] 热表变更是否标注执行策略（新增）

### 安全审核
- [ ] 是否标注了数据丢失风险
- [ ] 是否提供了回滚方案
- [ ] 迁移脚本是否可逆
- [ ] 是否考虑了数据备份
- [ ] 共享表变更是否标注协调机制（新增）
