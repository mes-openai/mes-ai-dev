# 约束规则

## 约束规则

### 必须遵守
1. **执行前输出计划**：在任何操作前必须输出执行计划
2. **结果审核**：输出前必须审核调用链路的准确性
3. **按需加载代码**：只读取涉及的代码文件，控制上下文预算
4. **不硬编码路径**：基于知识库索引定位代码文件
5. **上下文预算控制**：代码文件读取总量控制在 20-40K token
6. **文件交接**：通过 raw-requirement.md 和生成的报告文件传递信息

### 禁止行为
1. ❌ 不批量读取代码目录
2. ❌ 不读取未涉及的代码文件
3. ❌ 不猜测未在知识库中记录的调用关系
4. ❌ 不省略分支流程追踪
5. ❌ 不忽略异常处理流程

### 代码读取策略
```
策略1：基于知识库索引定位文件路径
  frontend-backend-map.md → 前端API文件路径
  api-registry.md → 后端Controller路径

策略2：逐层深入读取
  先读Controller → 确认调用的Service
  再读Service → 确认调用的DAO
  最后读DAO/Mapper → 确认SQL操作

策略3：遇到跨服务调用时
  读取 service-dependencies.md
  定位被调用服务的Controller/Service
  继续追踪该服务的调用链
```
