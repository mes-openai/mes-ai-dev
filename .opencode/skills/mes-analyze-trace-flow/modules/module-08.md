# 审核要点

## 审核要点

### 完整性检查
- [ ] 是否追踪了完整的调用链路（前端→后端→数据库）
- [ ] 是否识别了所有涉及的接口
- [ ] 是否标注了跨服务调用
- [ ] 是否梳理了异常流程
- [ ] 是否识别了 provider 选择点与架构允许路径
- [ ] 是否标注了仍需补证的断点和公共契约定义源

### 准确性检查
- [ ] API 路径是否与代码一致
- [ ] Controller 方法是否正确对应
- [ ] Service 调用关系是否准确
- [ ] 数据表操作是否完整

### 风险评估
- **调用链过长** → 可能存在性能问题
- **跨服务调用多** → 可能存在分布式事务问题
- **缺少异常处理** → 可能存在稳定性风险
