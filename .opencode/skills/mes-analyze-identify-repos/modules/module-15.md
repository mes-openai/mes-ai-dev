# 约束规则

## 约束规则

### 必须遵守
1. **执行前输出计划**：在任何操作前必须输出执行计划
2. **结果审核**：输出前必须审核仓库清单的准确性
3. **不硬编码仓库路径**：基于已有分析结果提取
4. **不读取所有配置文件**：只读取涉及的配置文件
5. **脱敏处理**：配置值涉及敏感信息需脱敏
6. **文件交接**：通过已有文件获取输入，不依赖对话历史

### 禁止行为
1. ❌ 不批量读取所有仓库目录
2. ❌ 不读取未涉及的配置文件
3. ❌ 不暴露敏感配置值（密码、密钥等）
4. ❌ 不省略风险评估
5. ❌ 不遗漏配置项变更

### 配置读取策略
```
策略1：基于服务列表读取配置
  影响的服务列表 → 读取对应的 application.yml

策略2：只读取必要的配置项
  数据库连接 → spring.datasource.*
  服务调用 → restService.properties
  中间件配置 → redis/mq配置

策略3：脱敏敏感信息
  密码 → [已脱敏]
  密钥 → [已脱敏]
  连接串 → [已脱敏]
```
