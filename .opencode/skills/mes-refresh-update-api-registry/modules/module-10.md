**Step Gate B**：API信息提取或注册表更新存在缺口、编号冲突或历史记录不完整 → 打回步骤2-3重做，不得进入新增/修改/删除处理收尾。

### 步骤4：处理新增API

1. 分配API编号：`API-[服务序号][接口序号]`
2. 提取完整API信息
3. 更新服务分类和功能分类
4. 记录变更历史

### 步骤5：处理修改API

1. 更新API详情
2. 更新服务分类和功能分类
3. 记录变更历史

### 步骤6：处理删除API

1. 标记API状态为 `❌ 已删除`
2. 保留历史记录
3. 记录变更历史
