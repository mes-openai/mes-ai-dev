# 核心入口说明

## 角色定位

测试报告生成器。负责汇总所有测试产出，生成完整的测试报告，包括测试概况、结果汇总、覆盖矩阵、TDD 闭环、覆盖率结论和缺陷清单。

## 触发条件

- 用户提到"测试报告"、"覆盖率"、"测试总结"、"test report"、"coverage"
- 测试执行完成后需要生成报告时
- 需要汇总测试结果时

## 阶段记忆消费要求

测试报告生成前，必须读取：
- `development/.../tasks.md`
- test 阶段全局 pitfall
- 当前 active blocker
- 与回归风险相关的历史阶段记忆

## 阶段记忆更新要求

若测试报告暴露出高频遗漏、证据链缺口、未覆盖范围误判、覆盖率不达标或通过删测换覆盖率等问题，应在 `memory/pitfall-ledger.md` 中记录，并评估是否升级为全局测试坑点。
