# 执行步骤（1/3）

## 执行步骤

### 1. 输出执行计划

在开始设计前，必须先输出本次执行计划：
- **目标**：明确本次前端设计的范围和目标
- **步骤**：列出将要执行的详细步骤
- **预期产出**：明确将生成的前端设计文档
- **风险评估**：识别可能的前端兼容性和性能风险

### 2. 读取前置输入

按以下顺序读取必要的输入文件：

1. **需求规格文档**
   - 路径：`mes-ai-dev/workspace/requirements/REQ-YYYYMMDD-XXX/spec.md`
   - 重点提取：UI需求、交互需求、功能需求

2. **技术方案文档**
   - 路径：`mes-ai-dev/workspace/designs/REQ-YYYYMMDD-XXX/tech-approach.md`
   - 重点提取：涉及的前端模块、技术选型

3. **前端总览**
   - 路径：`mes-ai-dev/knowledge/code-map/frontend-overview.md`
   - 用途：了解前端整体架构和模块划分

4. **模块详情索引**
   - 路径：`mes-ai-dev/knowledge/code-map/modules/module-xxx/detail.md`
   - 根据技术方案中涉及的模块，读取对应模块目录下的详细索引

5. **前后端映射**
   - 路径：`mes-ai-dev/knowledge/dependency-graph/frontend-backend-map.md`
   - 用途：了解前端API调用与后端服务的对应关系

6. **API设计文档**
    - 路径：`mes-ai-dev/workspace/designs/REQ-YYYYMMDD-XXX/api-design.md`
    - 用途：了解需要调用的后端接口

**Step Gate A**：前置输入不完整、模块范围不清或 API 基线不足 → 打回步骤2重做，不得进入前端架构分析。

### 3. 前端架构分析

基于前端总览和模块详情：
- 分析现有前端架构模式（Vue版本、状态管理、路由模式）
- 了解现有组件库和公共组件
- 确认路由配置规范
- 了解API调用层封装方式

### 4. 页面结构设计

#### 4.1 页面规划

为每个功能模块规划页面：

**页面基本信息**
- 页面名称
- 页面路由路径
- 页面功能说明
- 所属模块

**页面组成**
- 页面布局结构
- 组件划分
- 数据流向
- 交互行为
