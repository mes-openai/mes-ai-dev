# 7. Schema画像汇总

## 7. Schema画像汇总
```yaml
schema: mes_main
owner_service: mes-production
table_count: [数量]
risk_level: 中风险
key_risks:
  - 热表：work_order
  - 共享表：material
  - 大表：trace_log
```
```

### Step 5.1: 产物判定规则

- `index.md` 为所有 Schema 的标准必需产物
- `tables.md` 不是基础建图默认强制产物；仅在明确进入深化明细模式时强制生成
- `relations.md` 不是基础建图默认强制产物；仅在明确进入关系分析范围时强制生成
- 若本次未生成 `tables.md` / `relations.md`，必须在 `index.md` 中说明当前深度边界、未覆盖范围及后续补齐建议

### Step 5.5: 风险画像评估规则

> Schema风险等级的判定规则。

**热表判定**：
- 🔴多(>10)：超过10张表被高频访问（从日志或代码推断）
- 🟡中(4-10)：4-10张热表
- 🟢少(≤3)：不超过3张热表

**共享表判定**：
- 🔴多(>3)：超过3张表被多个服务直接访问
- 🟡有(1-3)：1-3张共享表
- 🟢无：无共享表（所有跨服务访问通过API）

**大表判定**：
- 🔴多(>3)：超过3张表数据量>100万行
- 🟡有(1-3)：1-3张大表
- 🟢无：无大表

**综合风险等级判定**：
- 🔴高风险：热表>3 或 共享表>3 或 有触发器 或 大表>3
- 🟡中风险：有热表(≤3) 或 有共享表(≤3) 或 有大表(≤3)
- 🟢低风险：无热表、无共享表、无触发器、无大表

**Step Gate B**：满足以下全部条件后方可进入注册片段生成：
- SQL 解析已完成，或已明确记录无法完整解析的原因
- Schema 分类已完成
- 风险画像已形成正式结论，不得仅停留在占位标题
- 关键对象摘要已形成，不得为空

否则打回步骤3-5.5重做，不得生成注册片段。

### Step 6: 生成数据库注册表
```
工具：Write
路径：mes-ai-dev/knowledge/database-index/schema-xxx/registry-fragment.md
```

**Step Gate C**：满足以下条件后方可视为数据库索引步骤完成：
- `index.md` 已生成且内容达标
- 若本次目标包含表结构明细，则 `tables.md` 已生成且非空壳
- 若本次目标包含关系分析，则 `relations.md` 已生成且非空壳
- 所有缺失项均已写明原因、影响范围与后续补齐建议

否则打回步骤6重做，不得交付下游初始化步骤。
