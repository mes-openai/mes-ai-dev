# 执行步骤

## 执行步骤

### 1. 执行前输出计划（必须）
输出本次分析计划，包含：目标、步骤、预期产出、风险。等待用户确认。

### 2. 读取历史需求
从 workspace/requirements/ 下定位历史需求文档：
- raw-requirement.md（历史需求要素）
- spec.md（历史需求规格主文档）

**Step Gate A**：历史需求或新需求输入读取不完整 → 打回步骤2-3重做，不得进入差异对比。

### 3. 读取新需求
当前需求的 raw-requirement.md

### 4. 功能点差异对比
逐条对比新旧需求的功能点：
- **新增**：新需求中存在但历史需求中没有的功能点
- **修改**：新需求中存在且历史需求中也存在，但描述或规则有变化
- **不变**：新需求中存在且历史需求中也存在，无变化
- **废弃**：历史需求中存在但新需求中不再提及

### 5. 影响差异分析
对比新旧影响范围：
- 新增涉及的服务/模块
- 不再涉及的服务/模块
- 影响程度变化

**Step Gate B**：功能点差异或影响差异分析存在缺口 → 打回步骤4-5重做，不得生成差异报告。

### 6. 生成差异报告
输出差异报告到 raw-requirement.md 的差异分析章节

**Step Gate C**：差异报告缺少新增/修改/不变/废弃分类、影响等级或增量建议 → 打回步骤6重做，不得交付后续需求分析步骤。
