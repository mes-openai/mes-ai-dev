# 约束规则

## 约束规则

### 执行前必须输出计划
每次执行前必须输出执行计划，经用户确认后才可继续。

### 必须读取现有风格
在编写任何脚本前，必须：
1. 读取目标Schema的现有脚本（至少3个）
2. 分析命名规范、注释风格、SQL格式
3. 保持与现有脚本风格一致

### 命名规范
- 表名：小写字母 + 下划线，如 `order_header`
- 字段名：小写字母 + 下划线，如 `order_number`
- 主键：`id`
- 外键：`fk_表名_字段名`
- 唯一索引：`uk_字段名`
- 普通索引：`idx_字段名`
- 组合索引：`idx_字段1_字段2`

### 注释要求
- 所有脚本必须有文件头注释（需求编号、功能描述、作者、时间）
- 所有DDL语句必须有块注释
- 所有表必须有表注释
- 所有字段必须有字段注释

### 安全性要求
- DDL脚本必须使用 `IF NOT EXISTS` 或 `IF EXISTS`
- DELETE/UPDATE必须带WHERE条件
- 大批量数据操作需分批处理

### 回滚脚本要求
- 每个DDL/DML操作必须有对应的回滚脚本
- 回滚脚本顺序与执行脚本相反
- 回滚脚本必须经过测试验证
