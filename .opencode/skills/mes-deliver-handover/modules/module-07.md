# 执行步骤（1/2）

## 执行步骤

### 1. 输出执行计划

```
本次执行计划：
目标：汇总全生命周期产出，生成交付交接文档
步骤：
  1. 读取所有前置阶段产出文件
  2. 汇总完整交付物清单
  3. 整理运维要点
  4. 整理回滚操作指南
  5. 列出后续待办事项
  6. 生成交付边界说明（delivery-scope.md）
  7. 生成交付交接文档
  8. 释放并发锁
预期产出：handover-doc.md, delivery-scope.md
风险：遗漏关键运维信息
```

### 2. 汇总交付物清单

按阶段汇总所有产出文件：

| 阶段 | 交付物 | 路径 | 状态 |
|------|--------|------|------|
| 需求分析 | 需求规格文档 | workspace/requirements/ | ✅ |
| 详细设计 | 设计文档 | workspace/designs/ | ✅ |
| 代码开发 | 代码文件 + 自审报告 | jalor/ + workspace/development/ | ✅ |
| 测试验证 | 测试报告 + 测试代码 | workspace/testing/ | ✅ |
| 发布交付 | 部署计划 + 验收报告 + 发布说明 | workspace/delivery/ | ✅ |

**Step Gate A**：交付物清单不完整或关键输入阶段产物缺失未标注 → 打回步骤2重做，不得进入运维要点整理。

### 3. 整理运维要点

从各文档中提取运维团队需要的关键信息：

#### 3.1 配置项清单
- 新增/修改的配置项
- 各环境配置值
- 敏感配置项的存储位置

#### 3.2 监控告警
- 需要关注的业务指标
- 需要添加的监控规则
- 告警阈值建议

#### 3.3 日志位置
- 新增/修改的日志文件路径
- 日志级别配置
- 关键日志关键字（用于排障）

#### 3.4 常见问题处理
- 已知问题及临时解决方案
- 常见错误及排查步骤
- 联系人信息

### 4. 整理回滚操作指南

从部署计划中提取回滚相关信息：

1. 回滚触发条件
2. 回滚步骤（按序）
3. 回滚验证方式
4. 数据回滚注意事项
5. 回滚后的监控要点

### 5. 列出后续待办事项

整理尚未完成的工作项：

| 待办项 | 类型 | 优先级 | 说明 |
|--------|------|--------|------|
| [具体事项] | 功能/优化/技术债务 | P0-P3 | [说明] |

来源：
- 验收报告中的遗留问题
- 测试报告中未修复的P2/P3缺陷
- 代码自审中的技术债务标记
- 需求规格中延期的功能点

**Step Gate B**：运维要点、回滚指南或后续待办存在缺口 → 打回步骤3-5重做，不得进入交付边界说明生成。

### 6. 生成交付边界说明

使用 `templates/deliver/delivery-package-template.md` 模板，生成交付边界说明文档：
1. 列出本次交付的功能点清单（已完成）
2. 标注未完成项及原因（延期/范围调整/技术阻塞）
3. 说明已知限制和规避方案
4. 列出后续迭代计划
5. 生成交付确认矩阵（各角色确认事项）
6. 产出写入：`mes-ai-dev/workspace/delivery/REQ-YYYYMMDD-XXX/delivery-scope.md`
