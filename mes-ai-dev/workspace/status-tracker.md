# 需求状态追踪器

> 本文件用于追踪所有需求的阶段进度、门禁结论与退回记录。
> 更新协议见：`knowledge/reference/status-tracker-governance.md`

---

## 状态定义

| 状态 | 含义 | 可转换到 |
|------|------|---------|
| `draft` | 需求已提交，未开始分析 | `analyzing` |
| `analyzing` | 需求分析中 | `designed` / `draft` |
| `designed` | 设计完成，待开发 | `developing` / `analyzing` |
| `developing` | 开发中 | `testing` / `designed` |
| `testing` | 测试中 | `delivering` / `developing` |
| `delivering` | 交付中 | `closed` / `testing` |
| `closed` | 已交付关闭 | — |
| `blocked` | 被阻塞 | 恢复原状态 |

## 使用规则

1. **阶段进入门禁通过后**：更新 `状态/当前阶段/当前步骤/最近门禁结论`
2. **步骤级门禁通过后**：更新 `当前步骤/最近门禁结论`
3. **`⚠️有条件通过`**：必须将 `条件闭环状态` 更新为 `待闭环`
4. **`❌不通过` 或退回**：必须更新 `退回原因`
5. **并行冲突检测**：检查 `涉及服务` 列，同一服务不可同时有两个 `developing` 状态的需求
6. **锁机制联动**：进入 `developing` 时创建锁，`closed` 时释放锁
