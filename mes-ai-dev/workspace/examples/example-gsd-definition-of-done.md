# GSD 完成定义（示例）

> 示例文件，演示如何为开发阶段的单个工作单元定义完整完成、GSD 完成与不完成状态。
> 示例路径场景：`mes-ai-dev/workspace/development/REQ-20260416-002/dod-report-submit-api.md`

---

## 一、基本信息

| 字段 | 内容 |
|------|------|
| 阶段/任务名称 | 报工提交接口开发 |
| 适用模式 | both |
| 适用范围 | 工作单元 |
| 定义时间 | 2026-04-16 20:35:00 |
| 定义人/Agent | Sisyphus |

---

## 二、当前目标

实现报工提交接口，使后端可接收报工请求、完成基本校验并返回统一响应结构。

---

## 三、完整完成定义

| 类别 | 内容 |
|------|------|
| 必达项 | Controller、Service、DTO、VO、Mapper 均已实现；验证路径完整；关键异常场景已覆盖；开发自审已完成 |
| 验证方式 | 代码实现存在；接口可调用；关键路径验证通过；自审报告存在 |
| 不通过条件 | 核心字段未落库；统一响应结构不一致；关键异常未处理 |

---

## 四、GSD 完成定义

| 类别 | 内容 |
|------|------|
| 必达项 | 接口主路径可调用；核心字段校验已形成；统一响应结构已稳定；风险与后补动作已记录 |
| 验证方式 | 冒烟调用通过；最小验证记录存在；blocker 与后补动作可追溯 |
| 不通过条件 | 接口主路径不可用；关键字段校验缺失；无法判断是否可提测 |

---

## 五、可后补项

| 项目 | 内容 |
|------|------|
| 可后补项 | 非关键异常分支测试、扩展日志字段 |
| 后补原因 | 当前不阻断最小可提测结论 |
| 后补截止条件 | 进入测试阶段前完成 |

---

## 六、结论规则

- 满足完整完成定义 → 标记完整完成
- 不满足完整完成但满足 GSD 完成 → 标记 GSD 完成
- 两者都不满足 → 标记不完成
