## 二、blocker 详情

| 项目 | 内容 |
|------|------|
| blocker 类型 | 外部依赖阻塞 |
| blocker 等级 | P2 |
| blocker 描述 | 前端页面最终字段显示顺序待产品确认，导致详情页展示稿无法定稿 |

## 三、推进判定

| 项目 | 内容 |
|------|------|
| 最小决策点 | 产品只需确认详情页字段显示顺序是否必须与现有页面完全一致 |
| 当前推进判定 | 局部推进 |
| 代偿动作 | 先按现有页面字段顺序生成最小设计包，并允许后续前端在不改接口的前提下微调展示顺序 |
| 后补动作 | 产品确认后补充 `frontend-design.md` 的最终交互稿 |
| 风险等级 | 中 |
| 风险说明 | 若产品最终要求不同顺序，前端页面需做轻量调整，但不影响后端与接口设计继续推进 |
