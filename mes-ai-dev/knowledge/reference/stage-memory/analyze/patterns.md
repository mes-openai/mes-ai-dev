## 二、模式条目

### PAT-ANALYZE-G-001 先索引后细读的影响范围收敛模式

- 阶段：analyze
- 类型：pattern
- 状态：active
- 适用场景：
  - 大仓 / 超大仓
  - 涉及多个候选服务/模块
- 模式描述：
  - 先读第 0 层总览和依赖图，确认候选范围
  - 再读第 1 层索引和相关历史 handoff
  - 最后按需进入 detail / file-summaries / 数据库索引
- 执行要点：
  - 必须先确认 scope，再深入明细
  - 必须结合历史阶段记忆判断是否已有已知坑点
