# 错误码注册表

> 由 `/mes-init-project` 阶段自动生成，用于汇总系统错误码规范、服务编号与触发条件。
> 本模板在初始化阶段由 `mes-init-extract-reference` 扫描异常类、错误码常量类与接口返回定义后补全。

## 一、当前状态

- 状态：未填充 / 占位态 / 已填充（按实际填写）
- 是否可消费：否 / 条件可消费 / 是（按实际填写）
- 是否已确认公共错误契约定义源：是 / 否 / 部分确认
- 来源类型：`workspace-source` / `repo-source` / `sources-jar` / `decompiled-jar` / `unknown`

## 二、最低事实要求

错误码注册表至少应补充：
- 错误码来源位置
- 是否 i18n key
- 是否存在扩展参数
- 消费方式
- 适用范围
- 真实样例
