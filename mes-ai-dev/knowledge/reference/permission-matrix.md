# 权限矩阵

> 定义各角色在 MES-AI-DEV 框架中的操作权限。

---

## 一、角色定义

| 角色 | 说明 |
|------|------|
| 产品经理 | 需求提交、验收确认 |
| 设计负责人 | 技术方案审批、设计文档审核 |
| 开发工程师 | 代码编写、自审 |
| 测试工程师 | 测试执行、缺陷管理 |
| 运维工程师 | 部署执行、环境管理 |
| 项目经理 | 全局审批、需求状态管理 |

## 三、人工审核门禁权限

| 门禁节点 | 谁可以确认 | 确认内容 |
|---------|-----------|---------|
| 需求分析完成 | 产品经理 + 设计负责人 | 需求规格是否完整 |
| 详细设计完成 | 设计负责人 | 技术方案是否可行 |
| 代码开发完成 | 设计负责人 + 开发工程师 | 代码质量是否达标 |
| 测试验证完成 | 测试工程师 + 产品经理 | 功能是否满足验收标准 |
| 部署执行前 | 运维工程师 + 项目经理 | 是否允许部署 |
| 交付签收 | 产品经理 + 项目经理 | 交付物是否完整 |

## 五、知识库操作权限

| 操作 | 权限 |

| 读取 knowledge/ | 所有角色 |
