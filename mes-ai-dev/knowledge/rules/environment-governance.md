# 环境治理细则

> 对应摘要：`knowledge/rules/environment-governance-summary.md`
> 若当前任务不涉及环境发布、配置治理、数据库变更执行或回滚判断，优先读取摘要并禁止默认整篇读取正文。
> 本文件为硬性规则（违反=缺陷），定义多环境（开发/测试/预发布/生产）的治理标准。

---

## 一、环境定义

| 环境 | 用途 | 数据源 | 配置管理 | 部署方式 |
|------|------|--------|---------|---------|
| DEV（开发） | 开发联调 | 开发库（可造数据） | 本地配置文件 | 手动部署 |
| TEST（测试） | 功能/集成测试 | 测试库（脱敏数据） | 配置中心-测试分支 | CI 自动部署 |
| STAGING（预发布） | 验收/UAT | 生产库镜像（脱敏） | 配置中心-预发布分支 | CI 自动部署 |
| PROD（生产） | 正式运行 | 生产库 | 配置中心-生产分支 | 审批后手动触发 |

---

## 二、环境配置管理规则

### 2.1 配置项分类

| 类别 | 示例 | 管理方式 | 修改权限 |
|------|------|---------|---------|
| 环境固定配置 | 端口、数据源URL | 配置中心按环境隔离 | 运维 |
| 业务可调配置 | 功能开关、阈值参数 | 配置中心动态下发 | 产品经理 |
| 安全敏感配置 | 数据库密码、API密钥 | 加密存储 | 安全管理员 |

### 2.2 配置变更检查清单

每次部署前必须检查：

| # | 检查项 | 验证方式 |
|---|--------|---------|
| 1 | 各环境配置项值已更新 | 对比配置中心各环境 |
| 2 | 新增配置项在所有环境都已添加 | 配置中心检查 |
| 3 | 敏感配置已加密 | 配置值不以明文存储 |
| 4 | 废弃配置项已清理 | 无遗留的过期配置 |

---

## 三、环境部署规则

### 3.1 部署顺序（强制）

```
DEV → TEST → STAGING → PROD
```

- **禁止跳级**：不允许从 DEV 直接部署到 PROD
- **禁止跨环境**：每个环境必须独立验证
- **冒烟测试**：每个环境部署后必须执行冒烟测试

### 3.2 各环境部署前检查

| 环境 | 必须满足的条件 |
|------|--------------|
| DEV | 代码编译通过 |
| TEST | DEV 环境冒烟测试通过 |
| STAGING | TEST 环境全部测试用例通过 |
| PROD | STAGING 环境 UAT 通过 + 人工审批 |

### 3.3 各环境部署后验证

| 环境 | 验证项 |
|------|--------|
| DEV | 服务启动成功（/health 返回 200） |
| TEST | 冒烟测试通过 + 核心业务流程可用 |
| STAGING | 全部 UAT 场景通过 + 性能指标达标 |
| PROD | 健康检查 + 冒烟测试 + 监控指标正常 |

## 四、数据库变更治理

### 4.1 变更分类

| 变更类型 | 示例 | 向下兼容 | 需要数据迁移 |
|---------|------|---------|-------------|
| 新增表 | 新建配置表 | 是 | 否 |
| 新增字段（有默认值） | ADD COLUMN with DEFAULT | 是 | 否 |
| 新增字段（无默认值） | ADD COLUMN NOT NULL | 否 | 是 |
| 修改字段类型 | INT → BIGINT | 视情况 | 视情况 |
| 删除字段 | DROP COLUMN | 否 | 是 |
| 删除表 | DROP TABLE | 否 | 是 |

### 4.2 执行规则

1. **不兼容变更必须分两阶段**：
   - 第一阶段：添加新字段/表（兼容旧代码）
   - 第二阶段：确认新代码部署后，再删除旧字段/表
2. **数据迁移脚本必须可回滚**
3. **每个环境的数据库变更必须记录在 db-migration.md**

---

## 五、环境回滚规则

### 5.1 回滚决策矩阵

| 场景 | 影响范围 | 回滚方式 | 审批 |
|------|---------|---------|------|
| 部署失败（服务未启动） | 单服务 | 自动回滚到上一版本 | 不需要 |
| 冒烟测试失败 | 单服务 | 自动回滚 | 不需要 |
| UAT 不通过 | 多服务 | 回滚到上一个稳定版本 | 技术负责人 |
| 生产异常（发现于 30 分钟内） | 多服务 | 紧急回滚 + 告警 | 运维负责人 |
| 生产异常（发现于 30 分钟后） | 多服务 | 评估后决定回滚或热修复 | 项目经理 |

### 5.2 回滚验证

回滚后必须验证：
1. 服务健康检查恢复
2. 核心业务流程可用
3. 数据一致性确认（如涉及数据回滚）
4. 监控指标恢复正常

---

## 六、强制执行规则

1. **部署顺序不可跳级**：违反 = 事故
2. **数据库不兼容变更必须分阶段**：违反 = 事故
3. **每个环境部署后必须验证**：未验证 = 未完成
4. **回滚方案必须在部署前确认**：无回滚方案 = 不允许部署
