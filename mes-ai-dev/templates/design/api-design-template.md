### 请求示例

#### 正常请求示例

**GET请求**：
```
GET /api/v1/users?page=1&size=10&name=admin
```

**POST请求**：
```json
POST /api/v1/users
Content-Type: application/json

{
  "username": "testuser",
  "email": "test@example.com",
  "roleIds": [1, 2]
}
```

**PUT请求**：
```json
PUT /api/v1/users/123
Content-Type: application/json

{
  "username": "updateduser",
  "email": "updated@example.com"
}
```

---

#### 异常请求示例（用于测试）

**参数缺失**：
```json
POST /api/v1/users
{
  "username": "testuser"
  // 缺少必填的email字段
}
```

**参数校验失败**：
```json
POST /api/v1/users
{
  "username": "test user with spaces",  // 不符合pattern规则
  "email": "invalid-email"              // 不符合email格式
}
```

```json

  "message": "success",
  "data": {
    "[请填写字段名]": "[请填写类型说明]"
  }
}
```

**分页响应结构**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
```

### 异常响应

#### 错误码设计

| 错误码 | HTTP状态 | 错误消息 | 错误场景 | 处理建议 |
|--------|----------|----------|----------|----------|
| [请填写错误码] | [请填写HTTP状态] | [请填写错误消息] | [请填写场景] | [请填写处理建议] |

示例：

| 错误码 | HTTP状态 | 错误消息 | 错误场景 | 处理建议 |
|--------|----------|----------|----------|----------|
| 400 | 400 Bad Request | 参数校验失败 | 请求参数不符合规则 | 检查参数格式 |
| 401 | 401 Unauthorized | 未授权访问 | 缺少认证信息 | 重新登录 |
| 403 | 403 Forbidden | 无权限访问 | 用户无此API权限 | 联系管理员分配权限 |
| 404 | 404 Not Found | 资源不存在 | 请求的资源ID不存在 | 检查资源ID |
| 409 | 409 Conflict | 资源冲突 | 用户名已存在 | 更换用户名 |
| 500 | 500 Internal Server Error | 服务器内部错误 | 服务端异常 | 等待修复或联系管理员 |
