# Agent 交接模板（统一版）

> 用于 `*-completion.md` 和 `*-results.md` 交接文件。
> 适用场景：Agent 完成任务单元或交付分析/审核结果后，向主控 Agent 交接。
> 行数要求：completion 类 ≤150行，results 类 ≤200行。

---

## 使用方式

根据场景选择变体：
- **completion 变体**：开发任务完成、服务/模块单元完成
- **results 变体**：分析报告、审核意见、测试结果、知识刷新结果

## 使用规则

1. **必须包含四节**：结论 / 风险 / 下一步 / 证据路径
2. **不得粘贴大段代码或日志**，只写关键摘要和路径
3. **部分完成**时：必须明确哪些对象已完成、哪些未完成
4. **不通过/待补证**时：必须说明影响范围和建议动作
5. **结论摘要**应可直接被主控Agent消费，减少过程性复述
6. **命中并发锁、步骤门禁或汇总判断场景时**：必须填写“锁状态与门禁状态”章节
7. **命中局部消费、降级推进或部分失败场景时**：必须填写“部分失败与知识消费口径”章节
8. **若本次交接用于阶段退出汇总**：应在“证据路径”中引用 `stage-output-report.md` 或等价阶段完成产物报告路径
9. **若本次交接涉及阶段切换**：应优先引用当前阶段 OpenSpec 主交接文档，并明确说明哪些 active 风险、active pitfall 与 blocker 需要传递到下一阶段
