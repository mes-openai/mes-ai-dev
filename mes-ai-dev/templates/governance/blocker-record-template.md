# 阻塞记录模板

> 本模板用于记录当前目标推进过程中的 blocker（阻塞项）。
> blocker 记录的目标不是“报问题”，而是明确阻塞类型、推进判定、代偿动作、后补动作与升级路径。
> 本模板适用于 Strict 与 GSD 两种模式；当命中 GSD 模式时，blocker 记录为强制产物之一。

---

## 一、基本信息

| 字段 | 内容 |
|------|------|
| 需求编号 | REQ-YYYYMMDD-XXX / EMG-YYYYMMDD-XXX / N/A |
| 阶段 | analyze / design / develop / test / deliver / refresh / emergency |
| 当前模式 | strict / gsd |
| 当前目标 | [当前要推进到什么状态] |
| blocker 编号 | BLK-001 |
| 记录时间 | YYYY-MM-DD HH:mm:ss |
| 记录人/Agent | [名称] |

---

## 二、blocker 详情

| 项目 | 内容 |
|------|------|
| blocker 类型 | 硬阻塞 / 软阻塞 / 外部依赖阻塞 |
| blocker 等级 | P0 / P1 / P2 / P3 |
| blocker 描述 | [具体问题，必须绑定当前目标] |
| 影响范围 | [影响阶段/模块/服务/文档/环境] |
| 当前是否阻断推进 | 是 / 否 / 局部阻断 |
| 当前是否可继续推进 | 允许 / 不允许 / 局部允许 |

> blocker 描述必须具体，不得使用“还有问题”“需要后续确认”“有风险”等模糊描述替代。

---

## 三、推进判定

| 项目 | 内容 |
|------|------|
| 最小决策点 | [若为硬阻塞，压缩成最小需要确认的问题] |
| 当前推进判定 | 停止推进 / 局部推进 / 可继续推进 |
| 代偿动作 | [如何继续推进] |
| 后补动作 | [后续补什么] |
| 风险等级 | 高 / 中 / 低 |
| 风险说明 | [说明继续推进或停止推进的风险] |

---

## 四、责任与升级

| 项目 | 内容 |
|------|------|
| 当前责任人/角色 | [角色/名称] |
| 升级对象 | [角色/人员] |
| 升级条件 | [什么条件下必须升级] |

## 五、闭环状态

| 项目 | 内容 |
|------|------|
| 当前状态 | 待处理 / 可继续 / 已关闭 / 已转移 |
| 关闭条件 | [何时算关闭] |
| 复查时间 | [时间] |

## 六、结论

- **当前 blocker 结论**：阻断 / 非阻断 / 外部依赖待协调
- **对当前目标的影响**：停止推进 / 局部推进 / 可继续推进
- **推荐下一步**：[下一步动作]
